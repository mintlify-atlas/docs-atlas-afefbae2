Vagrant is a command-line tool for building and managing virtual machine environments. It gives you a consistent, reproducible development environment that runs on your laptop or in the cloud — and works the same way for every member of your team, regardless of whether they use Linux, macOS, or Windows.

## What is Vagrant?

Vagrant provides easy-to-configure, reproducible, and portable work environments built on top of industry-standard virtualization technology. You describe your environment in a single file called a `Vagrantfile`, then Vagrant handles provisioning, networking, and synced folders for you. Machines run on top of providers like VirtualBox, VMware, or AWS. Industry-standard provisioning tools — shell scripts, Chef, Puppet, Ansible — can then automatically install and configure software inside the virtual machine.

## Why use Vagrant?

The core promise of Vagrant is eliminating the "works on my machine" problem. When your team shares a `Vagrantfile`, everyone creates their environment from the same configuration. There is no manual setup, no diverging dependencies, and no debugging environment differences.

## How Vagrant compares to other tools

Vagrant is not a configuration management tool like Chef or Puppet — it orchestrates virtual machines and delegates provisioning to those tools. It is not a full virtualization platform like VirtualBox or VMware either; instead, it sits on top of those providers and gives you a unified workflow for managing them.

| Tool | Purpose | Vagrant's role |
|---|---|---|
| VirtualBox / VMware | Run virtual machines | Provider that Vagrant manages |
| Chef / Puppet / Ansible | Configure software on a machine | Provisioner that Vagrant invokes |
| Docker | Run containerized workloads | Can be used as a provider or provisioner |
| Terraform | Provision cloud infrastructure | Complementary — Vagrant focuses on dev environments |

<Note>
  Vagrant is designed for development environments. For production infrastructure provisioning, consider [Terraform](https://www.terraform.io/).
</Note>
