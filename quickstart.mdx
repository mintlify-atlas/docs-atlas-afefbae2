This guide walks you through the core Vagrant workflow from scratch: writing a `Vagrantfile`, starting a virtual machine, logging in over SSH, and tearing it all back down. By the end you will have run your first development environment with Vagrant and understand the commands you will use every day.

Before you begin, make sure you have [Vagrant installed](/installation) and [VirtualBox](https://www.virtualbox.org/) available on your machine. VirtualBox is free and works on Linux, macOS, and Windows.

## The core workflow

<Steps>
  <Step title="Initialize your project">
    Create a new directory for your project and run `vagrant init` to generate a `Vagrantfile`:

    ```bash
    mkdir my-vagrant-project
    cd my-vagrant-project
    vagrant init hashicorp/bionic64
    ```

    This creates a `Vagrantfile` in the current directory configured to use the `hashicorp/bionic64` box — a minimal Ubuntu 18.04 image published by HashiCorp.

    The generated `Vagrantfile` looks like this:

    ```ruby
    Vagrant.configure("2") do |config|
      config.vm.box = "hashicorp/bionic64"
    end
    ```

    The `Vagrantfile` is the single source of truth for your environment. You commit it to version control so every teammate uses the same configuration.

    <Tip>
      The `-m` flag generates a minimal `Vagrantfile` without comments: `vagrant init -m hashicorp/bionic64`
    </Tip>
  </Step>

  <Step title="Start the virtual machine">
    Boot the machine with:

    ```bash
    vagrant up
    ```

    On the first run, Vagrant downloads the box from HashiCorp's box catalog. Subsequent runs use the cached copy.

    ```
    Bringing machine 'default' up with 'virtualbox' provider...
    ==> default: Box 'hashicorp/bionic64' could not be found. Attempting to find and install...
        default: Box Provider: virtualbox
        default: Box Version: >= 0
    ==> default: Loading metadata for box 'hashicorp/bionic64'
        default: URL: https://vagrantcloud.com/hashicorp/bionic64
    ==> default: Adding box 'hashicorp/bionic64' (v1.0.282) for provider: virtualbox
    ==> default: Booting VM...
    ==> default: Waiting for machine to boot. This may take a few minutes...
        default: SSH address: 127.0.0.1:2222
        default: SSH username: vagrant
        default: SSH auth method: private key
    ==> default: Machine booted and is ready!
    ```

    <Note>
      If you have another hypervisor like KVM or Hyper-V active, VirtualBox may fail to start the VM. See [installing Vagrant](/installation) for how to resolve hypervisor conflicts.
    </Note>
  </Step>

  <Step title="Connect over SSH">
    Log into the running machine with:

    ```bash
    vagrant ssh
    ```

    ```
    Welcome to Ubuntu 18.04.3 LTS (GNU/Linux 4.15.0-58-generic x86_64)

    vagrant@vagrant:~$
    ```

    You are now inside the virtual machine. The `/vagrant` directory is automatically synced with your project directory on the host — any files you place there are immediately visible on both sides.

    ```bash
    ls /vagrant
    ```

    ```
    Vagrantfile
    ```

    Type `exit` to leave the SSH session and return to your host machine.
  </Step>

  <Step title="Suspend or halt the machine">
    When you are done working, you have two options:

    **Suspend** saves the machine state to disk and stops it. Resume is fast because the machine does not need to reboot:

    ```bash
    vagrant suspend
    vagrant resume
    ```

    **Halt** performs a graceful shutdown:

    ```bash
    vagrant halt
    ```

    Run `vagrant up` again to restart a halted machine.
  </Step>

  <Step title="Destroy the machine">
    When you no longer need the environment, destroy it to free disk space and memory:

    ```bash
    vagrant destroy
    ```

    ```
        default: Are you sure you want to destroy the 'default' VM? [y/N] y
    ==> default: Destroying VM and associated drives...
    ```

    Destroying a machine does not delete your `Vagrantfile` or any files in your project directory. You can recreate the environment at any time with `vagrant up`.
  </Step>
</Steps>

## Understanding the Vagrantfile

The `Vagrantfile` uses Ruby syntax but you do not need to know Ruby to use it. The most common settings are:

```ruby
Vagrant.configure("2") do |config|
  # The box to use as the base image
  config.vm.box = "hashicorp/bionic64"

  # Forward port 8080 on the guest to 8080 on the host
  config.vm.network "forwarded_port", guest: 8080, host: 8080

  # Run a shell script to provision the machine on first boot
  config.vm.provision "shell", inline: <<-SHELL
    apt-get update
    apt-get install -y nginx
  SHELL
end
```

Every time you run `vagrant up` on a new machine, Vagrant runs the provisioner automatically. This ensures the environment is always configured the same way.

<Info>
  Vagrant supports multiple provisioners including shell scripts, Ansible, Chef, and Puppet. You can also combine them.
</Info>

## Common commands reference

| Command | Description |
|---|---|
| `vagrant init <box>` | Create a new `Vagrantfile` for the given box |
| `vagrant up` | Start and provision the machine |
| `vagrant ssh` | Open an SSH session into the machine |
| `vagrant halt` | Gracefully shut down the machine |
| `vagrant suspend` | Save state and pause the machine |
| `vagrant resume` | Resume a suspended machine |
| `vagrant reload` | Restart the machine and re-apply the `Vagrantfile` |
| `vagrant destroy` | Stop and delete the machine |
| `vagrant status` | Show the current machine state |
| `vagrant box list` | List all locally cached boxes |
