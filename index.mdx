Vagrant is a command-line tool for building and managing virtualized development environments. With a single configuration file called the Vagrantfile, you define your machine, and Vagrant handles provisioning, networking, and synced folders — so every team member runs identical environments on VirtualBox, VMware, Docker, or cloud providers.

## What is Vagrant?

Vagrant wraps industry-standard virtualization technology into a single, consistent workflow. You write one Vagrantfile that describes your development environment, commit it to version control, and every developer on your team gets the same machine with `vagrant up`.

## Get started in four steps

<Steps>
  <Step title="Install Vagrant and VirtualBox">
    Download and install [VirtualBox](https://www.virtualbox.org/) and then Vagrant for your operating system. See the [installation guide](/installation) for full instructions.
  </Step>
  <Step title="Initialize a project">
    Create a new directory, then run `vagrant init` to scaffold a Vagrantfile with a base box:
    ```bash
    mkdir my-project && cd my-project
    vagrant init hashicorp/bionic64
    ```
  </Step>
  <Step title="Start the machine">
    Boot your virtual machine. Vagrant downloads the box and provisions the environment automatically:
    ```bash
    vagrant up
    ```
  </Step>
  <Step title="SSH into the machine">
    Connect to the running machine over SSH:
    ```bash
    vagrant ssh
    ```
  </Step>
</Steps>

<Tip>
  Run `vagrant destroy` when you're done to free disk space. Your Vagrantfile persists, so you can always recreate the environment with `vagrant up`.
</Tip>
